watering_timer_1
  • write X > 0: start timer / initiate high priority (manual) watering.
  • write X < 0: start timer / initiate low priority (scheduled) watering (initiated by SM action), ignored if a high priority watering is already running.
  • write 0: stop timer / cancel high and low priority (manual and scheduled) watering.
  • read 0: watering not running.
  • read X > 0: high priority (manual) watering running, will end at report_timestamp + X.
  • read X < 0: low priority (scheduled) watering running, will end at report_timestamp - X.

schedule_config

True

True

hex

252

Format: 0x[IIWWHHMMDDDDAA],[...] up to 36 schedules

  • I: schedule id
  • W: weekdays bitmask, LSB = sunday (first day in week)
  • H: start hour: 0x00 - 0x18
  • M: start minute: 0x00 - 0x3B
  • DD: duration in minutes [low byte, high byte]: [0x00, 0x00] - [0xFF, 0xFF]
  • A: action_id (see enum action_id)

Note: The times are in local time.

Size = 7 bytes x 36 schedules = 252 bytes

Example:

id=01,weekdays=Mon-Thu,time=14:59,duration=300s,action=actor1 --> 0x01 1E 0E 3B 05 00 00

Default value = empty
