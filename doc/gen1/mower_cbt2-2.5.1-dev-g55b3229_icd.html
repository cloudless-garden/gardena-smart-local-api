  • -2^24+1 : low prio timer with unknown length (intentionally not -2^24, because it's reserved for infinite)
  • [-2^24+2 ; -1] : reserved for future use
  • 0 : switch off
  • [0; 2^24-1] : timer in seconds
  • 2^24 : reserved for future use
  • write 16777216: not allowed.
  • write X > 0: start timer / initiate timer.
    • IRS command: 0x0E, subcmd: 29 (start mower), override timer: max(1, X / 60)
  • write 0: stop timer / cancel timer.
    • IRS command: 0x0E, subcmd: 30 (park mower), park time: 0xFFFE (park until next timer)
  • write X < 0: not allowed.
  • read 0: timer not running.
  • read X > 0: timer running, will end at report_timestamp + X.
    • Note: A value is only reported at the start and the end of a manual mowing session. When requesting a value during a session, the reported value is rounded up to the next minute.
  • read -16777215: schedule running, running time unknown.

Format: 0x[IIWWHHMMDDDDAA],[...] up to 14 schedules

  • I: schedule id
  • W: weekdays bitmask, LSB = sunday (first day in week)
  • H: start hour: 0x00 - 0x18

Note: The times are in local time. There can only be 2 schedules per day.

Size = 7 bytes x 14 schedules = 98 bytes

settings_control

True

True

hex

32

empty

byte order: DDPCMSEFZddpcmsefz

Capital letters stand for "apply" bitmask to the corresponding setting. All bits have to be set for a setting to be applied

  • dd (big endian) = starting distance: 15 - 300 cm
  • p = drive past wire: 15 - 50 cm
  • c = sensor control: 0 = disabled, 1 = reserved, 2 = low cutting time (decrease at 50% above idle current, increase at 80%% above idle), 3 = medium (default, decrease at 30%, increase at 60% above idle), 4 = high cutting time (decrease at only at 20%, increase at 50% above idle), 5 = reserved.
  • m = mower house: 0 = not installed, 1 = installed
  • s = security level: low = 3, medium = 7, high = 63
  • e = eco mode: 0 = eco mode off, 1 = eco mode on
  • f = frost option switch: 0 = disabled, 1 = enabled
  • z = zone generator switch: 0 = disabled, other = enabled bit 0: 0 = disabled, 1 = enabled

Settings for the mower to be applied from the back-end. A bit mask selects the setting(s) to be applied. All or no bits in a byte have to be set. For future compatibility a longer buffer can be written where unknown settings are ignored by the firmware, the first half of the buffer is always the bit mask.

  • starting distance: Start distance when mowing from charging station. IRS: 0x04 0x81
  • drive past wire: Defines how much the mower drives across the boundary wire. IRS: 0x04 0x90
  • sensor control (aka Autotimer): Allows to adapt mowing time according to lawn growth rate. Specifies blade motor current threshold above idle for an increase or decrease of mowing time per day by 3h. IRS: 0x06 0x07
  • mower house: When active, mower avoids driving across the charging station in order to prevent a collision. IRS: 0x02 0x82
  • security level: IRS: 0x04 0x84 Security Functions. Levels:
    • Low = time lock on, PIN off, alarm off
    • Medium = time lock on, PIN on, alarm off
    • High = time lock on, PIN on, alarm on Available levels on mowers: P0.1: [Low, Medium]. P0.2 and P15: [Low, Medium, High]
  • Eco mode: Turn loop fields off when mower is in charging station. IRS: 0x02 0x8F
  • Frost: Mower will stay in charging station (parked on frost) if a frost condition exists. Periodic measurements will be taken to check if frost condition persists. IRS: 0x02 0x82
  • Zone generator: If this function is available on the mower the user can switch on/off the zone generator for mobile loops. IRS: 0x02 0xA4
  • Apply all settings
  • starting distance: 60 cm
  • drive past wire: 27 cm
  • sensor control: 3,
  • mower house: 1
  • security: 0
  • eco mode: 1
  • frost: 1
  • zone generator: 1
