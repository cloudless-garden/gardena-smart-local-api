hex

252

Format: 0x[IIWWHHMMDDDDAA],[...] up to 36 schedules

  • I: schedule id
  • W: weekdays bitmask, LSB = sunday (first day in week)
  • H: start hour: 0x00 - 0x18
  • M: start minute: 0x00 - 0x3B
  • DD: duration in minutes [low byte, high byte]: [0x00, 0x00] - [0xFF, 0xFF]
  • A: action_id (see enum action_id)

Note: The times are in local time.

Size = 7 bytes x 36 schedules = 252 bytes

Example:

id=01,weekdays=Mon-Thu,time=14:59,duration=300s,action=actor1 --> 0x01 1E 0E 3B 05 00 00

Default value = empty

power_timer

True

True

number

s

-16777215

16777216

1

  • [-2^24+1 ; -1] low prio timer in seconds
  • 0 : switch off
  • [0; 2^24-1] : high prio timer in seconds
  • 2^24 = infinite on
  • write X > 0: start timer / initiate high priority timer.
  • write X < 0: start timer / initiate low priority timer (initiated by SM action), ignored if a high priority timer is already running.
  • write 0: stop timer / cancel high and low priority (manual and scheduled) timers.
  • read 0: timer not running.
  • read X > 0: high priority (manual) timer running, will end at report_timestamp + X.
  • read X < 0: low priority (scheduled) timer running, will end at report_timestamp - X.

sun_data

True

True

hex

254

Format: 2-byte header followed by 84 3-byte blocks: 0x oooo [ xxxxxx ] * 84

oooo: signed big endian 16 bit number: half distance (in seconds) from GMT meridian to the target longitude. example: for Zürich this is ( 8.55 / 360 * 86400 / 2) = 1026 = 0x0402 -> [0x02 0x04] example: for Madrid this is (-3.70 / 360 * 86400 / 2) = -444 = 0xFE44 -> [0x44 0xFE] Could be outside of -12h...12h range. For example, for Kiribati islands this is +13:34. If instead these will be configured as -10:26 that will introduce an error of about a minute. xxxxxx: lookup point, containing one signed and one unsigned integers: 10-bit integer encoding (Equation of Time) * 15/32 14-bit integer encoding (day length) * 6/32 Example: if EoT is -801 seconds and day length is 50108 seconds then: -801 * 15/32 ~= -375 = 0b 1010001000 50108 * 6/32 ~= 9395 = 0x413f = 0b 10010010110011

   0b10010010110011_1010001000 = 0x92ce88 -> [0x88 0xce 0x92]

The lookup points are spaced exactly 1/84th of mean tropical year apart (31556925/84 seconds apart). The first point corresponds to the unix time 1640102340 (2021-12-21 15:59:00 GMT) as well as to all exactly same second of all previous and following tropical years. For the reference: at that time the Earth has almost reached the perihelion and the EoT is midway decreasing from the maximum of 16 minutes to the minimum of -14 minutes.

IMPORTANT NOTICE! Make the logitude and latitude coarce enough to protect the sensitive geo location data. Example: Zürich coordinates are (+8.5417, +47.3769). By applying float(round(x*20))/20 we get (+8.55, 47.40). Such rounding introduces an error of no more than 2.8 kilometers.

sun_schedule_config

True

True

hex

252

Format: repeating blocks of 7 bytes - up to 36 blocks

mmmm mmmm ..bb .MMM | mmmm mmmm ..bb MMMM | i aaa .ccc | vvvv vvvv VVVV VVVV

schedule start | schedule end |id aid cycle| repetition value

  • b: offset (B)ase (00=midday, 01=sunrise, 10=sunset)
  • m: offset (M)inutes (signed 12 bit)
  • i: extra_id flag
  • a: (A)ction ID
  • c: repetition (C)ycle type ( 000=12 day cycle, 001=14 day cycle, 010=15 day cycle)
  • v: repetition (V)alue, i.e. days (12, 14 or 15 lower bits are used, bit 0 = first day)

PAY ATTENTION! all values are little endian. Uppercase letters denote upper bits.
